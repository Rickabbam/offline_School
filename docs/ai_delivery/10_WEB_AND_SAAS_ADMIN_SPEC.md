# WEB AND SAAS ADMIN SPEC

## Web app
School management/admin surface:
- settings
- dashboards
- reports
- user/admin tasks
- possibly admissions/reporting workflows best suited to browser

## SaaS admin
Platform operations surface:
- tenant provisioning
- school status
- subscription/status management
- controlled support access
- support audit visibility

## Guardrails
- platform data separation from school operational data
- impersonation is audited and limited
- no bypass of school/campus scoping through admin screens
