# DESKTOP APP SPEC

## Role
Primary campus operations client.

## Required workspaces
- Auth / trusted-device entry
- Onboarding wizard
- Admissions
- Students
- Staff
- Attendance
- Finance
- Exams / results / report cards
- Reports
- Settings
- Sync center / diagnostics / conflicts / recovery

## Desktop requirements
- Windows-first
- all critical school operations available offline
- local search
- queue/sync visibility
- printer support for receipts and report artifacts
- backup/export/restore operator flows

## Definition of done
Desktop is production-ready when:
- all MVP school operations can be completed offline
- sync behavior is visible and safe
- restore and recovery are operator-usable
- no scope leakage occurs across tenant/school/campus
